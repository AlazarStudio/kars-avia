**Kars Avia - История обновлений**

### **v0.1 (02.12.2024)**
- Реализован функционал управления профилем, включая возможность редактирования профиля и кнопку выхода.
- Обновлен раздел "О компании" для отелей и авиакомпаний.

### **v0.2 (04.12.2024)**
- Доработана фильтрация по авиакомпаниям и аэропортам.
- Улучшен механизм отмены заявок.
- Внесены исправления в работу модуля шахматки.
- Добавлена возможность обновления номерного фонда с загрузкой описания и изображений.
- В раздел "Об авиакомпании" добавлен подраздел с отображением номеров.

### **v0.3 (07.12.2024)**
- Реализовано сохранение параметра `isAirline` в локальном хранилище для корректного отображения отчетов.
- Оптимизирована работа с выпадающими списками в `DropdownList`.
- Расширено использование `DropdownList` в интерфейсе.

### **v1.0 (12.12.2024)**
- Внедрены обновления на сайт.
- Переработан интерфейс бокового меню: добавлена компактная версия и соответствующий мини-логотип.
- Улучшен поиск, обеспечена корректная работа с пагинацией, а также доработана фильтрация заявок.
- Оптимизированы всплывающие подсказки.

### **v1.1 (18.12.2024)**
- В случае совпадения дат заявок отображается уведомление с соответствующей ссылкой.
- Оптимизирован поиск по номерному фонду с учетом количества мест.

### **v1.2 (21.12.2024)**
- Доработан механизм редактирования отдела при изменении сотрудника авиакомпании.
- Добавлена возможность удаления отелей для диспетчера.

### **v2.0 (09.01.2025)**
- Доработаны фильтры статусов заявок.
- Исправлены ошибки, возникающие при архивации заявок.
- Улучшен механизм загрузки изображений: установлен лимит в 2 МБ.
- Реализована пагинация в модулях авиакомпаний и отелей.
- Обеспечена корректная работа скролла при навигации по страницам.
- Исправлены ошибки в отображении боковой панели в шахматке.

### **v2.1 (15.01.2025)**
- Оптимизирован механизм подписки на события.
- Доработан алгоритм отображения звездности отелей в соответствии с государственной оценкой.
- Исправлены ошибки в отображении тарифов номеров.

### **v2.2 (22.01.2025)**
- Добавлена поддержка двух чатов в заявках (для гостиниц и авиакомпаний).
- Реализована возможность логирования данных для авиакомпаний, отелей и резерва.
- Расширен функционал по добавлению номеров различной вместимости (от 5 до 8 мест).
- Начата работа над системой уведомлений, реализована первичная визуальная часть.

### **v3.0 (29.01.2025)**
- Внедрены обновления на сайт.
- Обеспечен доступ для авиакомпаний.
- Оптимизирована работа с архивными заявками, исключено их отображение в шахматке.
- Исправлена ошибка смены статусов через модальное окно в шахматке.
- Исправлены ошибки в разделении чатов между авиакомпаниями и гостиницами.
- Обеспечена адаптация интерфейса для небольших ноутбуков.

### **v3.1 (31.01.2025)**
- Реализовано разделение чатов у отелей в резерве, добавлены кнопки для каждого отеля и отображение названия отеля в чате.
- Продолжена работа над визуальной частью системы уведомлений.
- Доработан механизм редактирования дат заезда и выезда до момента заселения первого пассажира из резерва.
- Внесены исправления в `SVG`-разметку (`stroke-linecap`, `stroke-linejoin`, `clip-path`).
- Скорректирована логика отображения заявок в резерве для отелей.
- Исправлена ошибка с очисткой поля продления в `ExistRequest`.

### **v4.0 (04.02.2025)**
- Исправлена ошибка сохранения данных при переходе между вкладками в авиакомпаниях и экипаже.
- Устранена ошибка внесения цен для номеров вместимостью от 5 мест и выше.
- Исправлены проблемы с авторизацией авиакомпаний.
- Улучшена адаптация интерфейса для авиакомпаний и корректировка стилей цен гостиниц.
- Обновление подготовлено для тестирования авиакомпаниями.
- Восстановлена логика резерва с учетом `matchesCity`.
- Реализована поддержка цен для авиакомпаний.

### **v4.1 (05.02.2025)**
- Доработана адаптация интерфейса: реализовано отслеживание ширины экрана и состояния бокового меню.
- Реализовано скрытие реквизитов гостиниц от авиакомпаний.
- Во вкладке "О гостинице" контактная информация гостиниц заменена на контактные данные Kars Avia.
- Реализована возможность просмотра изображений номеров.
- Оптимизированы чаты в резерве и шахматке.

### **v4.2 (14.02.2025)**
- Внесены корректировки в соответствии с обновленной структурой базы данных.
- Улучшена адаптация интерфейса и проверка подписок.
- Реализованы подписки для шахматки авиакомпаний.
- Исправлены ошибки в шахматке отелей, устранены пересечения заявок при изменении дат через модальное окно.
- Доработана логика пересечения пассажиров и эскадрильи.
- Исправлен баг с отображением списка пассажиров: реализован корректный скроллинг и отображение элементов.

### **v4.3 (20.02.2025)**
- Заменен `DropdownList` на `Autocomplete` из Material UI на страницах с эстафетой и при создании заявки.
- Исправлена ошибка, препятствующая изменению авиакомпании.
- Добавлен индикатор загрузки (`Loader`) из Material UI во все модули с асинхронной загрузкой данных.
- Реализованы уведомления после создания и редактирования данных.
- В разделе "О гостинице" доработан интерфейс слайдера номеров: затемнены стрелки на первом и последнем слайде.

### **v5.0 (23.02.2025)**
- В Эскадрилью при создании заявки добавлено поле определяющее является заявка квотной или резервной (сбой).
- Изменен цвет сообщений собеседника в чате.
- Для авиакомпаний добавлены запросы на изменение даты, которые отправляются в чат.
- Добавлено выделение для запросов на изменение даты в чатах заявок в эскадрильи и пассажиров.
- Редактирование даты у пассажиров для диспетчеров доступно всегда.


### **v5.1 (25.02.2025) **
- В пассажирах при создании заявки есть возможность добавить манифест.
- Проверены подписки при создании и изменении сотрудников.
- Если манифест не был добавлен при создании, то можно добавить позже и также можно его обновить.
- Чат с гостиницами в пассажирах появляется только если добавить их. Визуально чата нет, пока не добавлена гостиница или гостиница не выбрала какое количество гостей она может разместить.
- Исправлен баг с чатами отелей, при добавлении новых отлей, все отели были в одном чате.
- `border-radius` везде увеличен с 5px до 10px.
- При редактировании пользователей изначально input`ы заблокированы, возможность редактировать появляется после нажатия изменить, а также добавлена возможность скрыть пароль звездочками (*).
- В разделах "О гостинице" и "О авиакомпании" визуальная составляющая для НЕ админов этих организаций и диспетчера изменена, input`ы заменены на обычные строки. 
- В отчетах для авиакомпаний есть возможность выбирать сотрудника, чтобы получить отчет по конкретному сотруднику.
- В эскадрильи проверено своевременное отображение истории (была проблема с подпиской).
- Редактирование профиля через настройки в процессе изменения.
